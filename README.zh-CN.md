## `>_` 它解决什么

很多人能分辨「好看」，却**说不清自己想要什么风格**，更不知道怎么落地。现有工具要么要求你已经知道要找什么（Mobbin / Refero），要么只是好看图片的海洋（Pinterest / Godly）。

Styledex 把**命名风格**当入口，把**可操作的风格配方**当留存价值——给独立开发者、初级设计师、创业者、爱好者一个「先用眼睛认出，再拿到能直接用的配色/排版/prompt」的工具。

## `>_` 核心特性

```
SUBJECT // 60 命名风格    极简 · 赛博朋克 · 瑞士国际主义 · Y2K · 浮世绘 · 便当网格 …
AXIS_01 // 配色           调色板可复制 HEX
AXIS_02 // 排版(一等维度)  对齐/字阶/密度/网格/字形 —— 可「按版式找风格」
LIVE    // 排版天性可交互  点选即改，描述/缩略图/放大详图实时重排
SAMPLE  // 跨品类 × 6      网页/App/海报/包装，每类 6 种版式，可放大轮播
BOARD   // 收藏夹          本地持久化
PROMPT  // 可带走          每个风格自带喂给生图/编程 AI 的提示词
VISUAL  // 赛博终端 HUD    扫描线 · 霓虹描边 · 电流马赛克干扰 · 一键 FX 开关
LANG    // 双语            中文 / English 一键切换
```

> **核心资产是风格配方 + prompt，而非图片库** —— 样本均由 CSS 程序化生成，零图片版权包袱，也因此特别适合社区共建。别人抄得了图片墙，抄不走一套打磨好的「配色组合 + 排版组合」映射。

## `>_` 贡献

最需要的贡献是 **新增设计风格**——加一个风格基本就是加一段结构化数据。完整步骤见 [CONTRIBUTING.md](./CONTRIBUTING.md)。目标：把风格库从 60 种，靠社区众包到几百种，做成一份开放的「设计风格图鉴」。

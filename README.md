## `>_` The problem

Plenty of people can tell what looks good but can't say *what style they want* — let alone how to build it. Existing tools either assume you already know what you're looking for (Mobbin, Refero) or are just an ocean of pretty pictures (Pinterest, Godly).

Styledex makes **named styles** the entry point and an **actionable style recipe** the payoff — so indie developers, junior designers, founders, and enthusiasts can go from "I'd know it if I saw it" to "here's the palette, type, and prompt I need."

## `>_` Features

```
SUBJECTS // 60 named styles    Minimalism · Cyberpunk · Swiss · Y2K · Ukiyo-e · Bento Grid …
AXIS_01  // Color              copy-paste HEX palettes
AXIS_02  // Typography (1st-class)  align / scale / density / grid / case — search styles BY layout
LIVE     // Interactive type   click a typography axis → narrative + previews re-flow live
SAMPLES  // Cross-category × 6 web / app / poster / packaging, 6 layouts each, zoom carousel
BOARD    // Favorites          save styles locally
PROMPT   // Take it with you   every style ships an AI prompt for image / coding agents
VISUAL   // Cyber terminal HUD scanlines · neon · interference glitch · one-tap FX toggle
LANG     // Bilingual          中文 / English toggle
```

> **The core asset is the style recipe + prompt, not an image library** — every sample is generated procedurally in CSS, so there are zero image-licensing headaches and the catalog is ideal for community curation. Anyone can copy an image wall; nobody can copy a well-curated "palette + typography" mapping.
